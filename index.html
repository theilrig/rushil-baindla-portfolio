03 · Project 1

Medical AI · AP Computer Science A · 2024–2025

Parkinson’s Disease
Detection via
Machine Learning

Course: AP Computer Science A & Independent Research · September 2024 – November 2025

🏆 Congressional App Challenge Winner — California District 10 · Presented at U.S. Capitol
ML model architecture:
Voice recordings → Acoustic feature extraction (Praat) → Random Forest classifier → Parkinson’s risk probability output → Flutter mobile app

System architecture — Parkinson’s detection pipeline

Project Objective

The objective of this project was to develop a machine learning system capable of detecting early-stage Parkinson’s disease using non-invasive voice recordings and wrist-based tremor measurements. Early detection of Parkinson’s disease is a critical clinical challenge — symptoms often go unrecognized until significant neurological damage has already occurred. This project aimed to create an accessible, low-cost diagnostic tool deployable as a mobile application, enabling patients and caregivers to monitor neurological risk indicators from home without requiring specialized clinical equipment.

My Responsibilities

I was the sole developer on this project, responsible for the complete research and engineering pipeline. My responsibilities included sourcing and preprocessing voice and tremor datasets, extracting acoustic features using Praat software (jitter, shimmer, harmonic-to-noise ratio), engineering and training a Random Forest classifier using Python and Scikit-learn, designing the PostgreSQL database schema for backend storage, and building the cross-platform mobile application in Flutter (Dart). I also prepared and delivered the full project presentation at the United States Capitol after winning the Congressional App Challenge for California’s 10th District in November 2025.

What I Learned & Skills Exemplified
End-to-End ML Development
I learned how to take a machine learning project from raw data collection through model training, evaluation, and production deployment — including strategies for handling class imbalance, feature selection, and cross-validation in a real medical dataset.
Acoustic Signal Processing
Using Praat, I learned to extract clinically relevant acoustic features from voice recordings — including jitter, shimmer, and harmonic-to-noise ratio — which are established biomarkers for Parkinson’s-related vocal tremor detectable before motor symptoms fully emerge.
Full-Stack Application Design
I developed proficiency in Flutter for cross-platform mobile development and PostgreSQL for backend storage, learning how to architect a complete application that integrates a trained ML model with a clean, usable patient-facing interface.
Science Communication
Presenting at the U.S. Capitol required translating complex technical work into clear, accessible language for a non-technical audience that included congressional staff and policymakers — a skill as important as the technical work itself.
Technologies & Tools Used
Python Scikit-learn Random Forest Praat Flutter PostgreSQL Dart Acoustic Signal Processing

04 · Project 2

Sleep Stage Classification
from EEG/EMG
via Deep Learning

Course: AP Research & Independent Study · January 2025 – Present

📄 Manuscript Currently Under Peer Review
Model benchmarking visualization:
LSTM · GRU · RNN · Transformer performance compared across five sleep stages (Wake / N1 / N2 / N3 / REM) vs. zero-shot LLM baseline

Architecture benchmark — sleep stage classification accuracy

Project Objective

The objective of this project was to develop and rigorously benchmark deep learning models for automated classification of sleep stages from polysomnography (PSG) recordings — specifically EEG (electroencephalography) and EMG (electromyography) time-series signals. Accurate, automated sleep staging has significant clinical value: manual scoring by sleep specialists is expensive and time-consuming, creating barriers for both diagnosis and large-scale research. This project evaluated four neural architectures — LSTM, GRU, vanilla RNN, and Transformer — against zero-shot large language model baselines, with the goal of identifying the most effective approach for this classification task and contributing novel findings to the published literature.

My Responsibilities

This is a solo independent research project conducted under AP Research. I was responsible for every stage of the work: sourcing publicly available EEG/EMG datasets, preprocessing raw time-series signals (bandpass filtering, artifact removal, epoch windowing, normalization), implementing all four model architectures from scratch in Python, designing a consistent benchmarking protocol with appropriate evaluation metrics, and writing the full research manuscript, which is currently under peer review. I also designed the experimental framework comparing trained deep learning models to large language models used as zero-shot classifiers — a novel contribution not widely explored in the sleep-staging literature.

What I Learned & Skills Exemplified
Deep Learning Architecture
I gained hands-on experience implementing LSTM, GRU, RNN, and Transformer architectures for sequential time-series data, developing an understanding of when each is appropriate and how they differ in capturing temporal dependencies at different scales.
Biomedical Signal Processing
I learned how to work with raw physiological signals — applying bandpass filtering, artifact removal, epoch segmentation, and normalization to EEG and EMG recordings to prepare them for machine learning analysis.
Research Methodology
Conducting publishable research required designing a rigorous experimental protocol, selecting appropriate evaluation metrics (accuracy, F1-score, Cohen’s kappa), and writing a scientific manuscript that meets the standards expected for peer review.
LLM Evaluation
Comparing trained neural models to zero-shot LLM baselines deepened my understanding of where large language models succeed and fail on structured, domain-specific tasks — and what it means to evaluate AI systems with genuine rigor and fairness.
